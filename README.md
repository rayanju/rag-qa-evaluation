# Retrieval-Augmented Question Answering

A modular Retrieval-Augmented Generation pipeline for question answering on the Stanford Question Answering Dataset (SQuAD).

The project compares lexical retrieval, dense retrieval using BERT embeddings, and Reciprocal Rank Fusion (RRF), measuring how each retrieval strategy affects downstream language-model performance.

## Pipeline

```text
Question
   │
   ▼
Preprocessing
   │
   ▼
Retriever
   ├── Token-overlap retrieval
   ├── Dense BERT retrieval
   └── Reciprocal Rank Fusion
   │
   ▼
Prompt construction
   │
   ▼
Language model
   │
   ▼
Answer
   │
   ▼
Evaluation
```

## Experiments

The notebook evaluates:

- Vanilla question answering without retrieval
- Oracle question answering using the ground-truth context
- Token-overlap retrieval
- Dense retrieval using BERT embeddings and cosine similarity
- The effect of retrieval depth
- Hybrid retrieval using Reciprocal Rank Fusion

Performance is measured using Exact Match, token-level F1, and ROUGE-2 F1.

## Results

| Retrieval strategy | Retrieved contexts | Exact Match | F1 Score | ROUGE-2 F1 |
|---|---:|---:|---:|---:|
| Token Overlap | 5 | 27.20% | 37.40% | 20.00% |
| Dense Retrieval | 10 | 23.60% | 33.77% | 16.67% |
| **Reciprocal Rank Fusion** | **5** | **32.80%** | **45.25%** | **22.90%** |

Reciprocal Rank Fusion produced the strongest results by combining the complementary rankings generated by lexical and dense retrieval. Compared with the best token-overlap configuration, RRF improved Exact Match by 5.6 percentage points and F1 by 7.85 percentage points without retraining the language model.

## Technical Approach

### Token-overlap retrieval

Candidate passages are ranked according to the number of normalized tokens shared with the question.

### Dense retrieval

Questions and candidate contexts are encoded using BERT. Contexts are ranked using cosine similarity between question and passage embeddings.

### Reciprocal Rank Fusion

RRF combines lexical and dense rankings using:

$$
RRF(d)=\sum_{r\in R}\frac{1}{c+\mathrm{rank}_r(d)}
$$

This avoids directly combining lexical-overlap and cosine-similarity scores, which operate on different scales.

## Context-Length Experiment

Retrieval depth was evaluated for \(k=1\), \(k=5\), and \(k=10\).

The language model supports a maximum input length of 4,096 tokens. A representative prompt containing 20 retrieved contexts required 4,638 tokens, so \(k=20\) could not be evaluated without truncating part of the evidence.

## Running the Project

The notebook was developed in Google Colab with GPU acceleration.

1. Open `notebooks/rag_qa_evaluation.ipynb`.
2. Select **Open in Colab** or upload it to Google Colab.
3. Enable a GPU runtime.
4. Run the notebook cells in order.

Model weights, generated embeddings, and intermediate evaluation files are not stored in this repository and are regenerated by the notebook.
